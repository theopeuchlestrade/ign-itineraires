# Code of Conduct

IGN Itinéraires aims to be a focused, respectful open source project.

## Expected Behavior

- Be respectful and constructive.
- Assume good intent, but be precise when discussing technical problems.
- Keep discussions relevant to the issue, pull request, or project decision.
- Accept maintainer decisions once a direction has been explained.
- Credit others' work when building on it.

## Unacceptable Behavior

- Harassment, insults, threats, or discriminatory language.
- Personal attacks or repeated off-topic arguments.
- Publishing private information without permission.
- Sexualized content or attention in project spaces.
- Coordinated disruption, spam, or abuse of GitHub issues, pull requests, or
  discussions.

## Scope

This code of conduct applies to project spaces, including issues, pull
requests, reviews, discussions, and maintainer-controlled community channels.
It also applies when someone is clearly representing the project.

## Reporting

Report conduct concerns to the maintainer through an established private
channel. Do not use public issues for private safety reports.

Security vulnerabilities should be reported through `SECURITY.md`, not through
this code of conduct process.

## Enforcement

Maintainers may remove comments, close discussions, reject contributions,
temporarily restrict participation, or permanently ban participants when needed
to keep the project healthy.

Enforcement should be proportional to the behavior, the impact on others, and
any repeated pattern.
