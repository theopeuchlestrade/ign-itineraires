# Governance

IGN Itinéraires uses lightweight maintainer-led governance.

## Maintainer

The current maintainer is `@theopeuchlestrade`.

The maintainer is responsible for:

- repository direction and scope;
- accepting or rejecting pull requests;
- release decisions;
- security response;
- dependency, privacy, and build policy;
- final decisions when consensus is not reached.

## Contribution Model

Contributions are welcome when they fit the project scope and quality bar.

For non-trivial work, open an issue or draft pull request first. This avoids
large changes being built against assumptions that may not match the project
direction.

## Decision Making

Technical decisions are discussed in issues and pull requests. The preferred
outcome is rough consensus backed by clear reasoning, privacy awareness, tests,
and maintainable implementation.

When tradeoffs remain, the maintainer makes the final call.

## Releases

Releases are cut by the maintainer. Versioning, changelog updates, deployment,
mobile signing, store publication, and rollback decisions remain maintainer
responsibilities.

## Becoming a Maintainer

Additional maintainers may be added after sustained, high-quality
contributions and demonstrated care for maintenance, privacy, security, and
review quality. There is no automatic promotion path.

## Policy Changes

Governance, support, conduct, security, privacy, and brand policies may evolve
as the project grows. Policy changes should be reviewed like other project
changes.
