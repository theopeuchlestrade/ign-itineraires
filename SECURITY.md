# Security Policy

## Supported Versions

Security fixes primarily target:

- the `main` branch;
- the official web deployment once it is released;
- official Android and iOS builds when they are distributed.

Older branches, unmaintained forks, and independently derived builds are not
guaranteed.

## Reporting a Vulnerability

Do not create a public issue to report a security flaw.

Use GitHub Private Vulnerability Reporting for this repository. If the feature
is temporarily unavailable, do not send sensitive details through a public
issue; wait for the private reporting channel to be restored. Avoid public
disclosure before a fix or mitigation is validated.

## What to Include

Please include, when possible:

- the affected platform, component, version, or commit;
- the expected impact and exploitation prerequisites;
- reproduction steps using public or synthetic locations;
- a minimal proof of concept;
- whether location, route history, external navigation, or network boundaries
  are affected.

## Disclosure Expectations

The maintenance goals are to acknowledge receipt, validate the report, prepare
a fix or mitigation, and coordinate disclosure once the risk is reduced. No
response-time or remediation-time service-level agreement is offered.
