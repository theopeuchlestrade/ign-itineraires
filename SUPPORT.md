# Support

This project is open source, but it does not include guaranteed support or
service-level commitments.

## Questions and Usage Help

Use GitHub issues for:

- reproducible bugs;
- documentation problems;
- focused feature requests;
- setup problems that appear to affect the project rather than one local
  environment.

Before opening an issue, check the README, existing issues, and current
documentation. Never publish a private home address, route, or precise
location; reproduce location problems with public landmarks or synthetic data.

## Security Reports

Do not report vulnerabilities through public issues. Follow `SECURITY.md`.

## Public Services

IGN Itinéraires depends on public Géoplateforme and map services. The project
cannot guarantee their availability, accuracy, quotas, compatibility, or
continued operation. It is not intended for emergency or safety-critical use.

## Builds and Self-Distribution

The Flutter app can be built for web and mobile. Maintainers do not provide
operational support for unrelated deployments or independently distributed
builds.

If you distribute a fork, you are responsible for:

- replacing protected IGN Itinéraires branding and application identifiers;
- mobile signing, hosting, and store compliance;
- privacy notices and legal obligations;
- compliance with third-party service and attribution requirements.

## Maintainer Availability

Issues and pull requests are reviewed on a best-effort basis. Clear
reproduction steps, focused patches, and passing tests make review faster.
