# Brand and Asset Policy

This repository's source code is licensed under `EUPL-1.2`. That licence
applies to the covered source code; it does not grant a general right to use
the IGN Itinéraires name, logos, app icons, domain names, screenshots, product
copy, or other brand assets.

This policy makes the project easier to fork and contribute to without
creating confusion about what is official.

The source-code copyright holder is Théo Peuchlestrade. IGN Itinéraires is the
project and product brand governed separately by this policy.

## Project Brand Assets

IGN Itinéraires brand assets include, without limitation:

- the IGN Itinéraires name and visual identity;
- app icons and launcher icons under `assets/branding/`, `web/`, `android/`,
  and `ios/`;
- screenshots, store assets, and marketing copy;
- domain names and public service names associated with the official project.

## Allowed Use

You may use the project names and repository references to:

- describe this project accurately;
- link to the original repository;
- say that a project is a fork of or derived from IGN Itinéraires;
- include screenshots or logs when reporting bugs, submitting pull requests,
  or documenting development work.

These uses must not imply endorsement, sponsorship, partnership, or official
status unless explicitly agreed.

## Restricted Use

Without explicit permission, do not:

- use IGN Itinéraires names, logos, icons, or visual identity as the primary branding
  for a public fork, hosted service, package, app-store listing, or commercial
  offering;
- register confusingly similar domains, accounts, package names, bundle
  identifiers, or application identifiers;
- present a modified build as the official IGN Itinéraires application;
- use brand assets in a way that suggests endorsement or partnership.

## Forks and App Builds

Public forks and independently distributed builds should use their own name,
icons, visual identity, domains, package names, bundle identifiers, and
application identifiers. It is acceptable to state factually that a project is
based on IGN Itinéraires when users can clearly distinguish it from the
official project.

For permission requests or unclear cases, contact the maintainer through the
